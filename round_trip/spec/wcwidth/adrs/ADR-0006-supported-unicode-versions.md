# ADR-0006: Supported Unicode Version Set

## Status
Accepted

## Context
Unicode has many historical versions. Supporting all versions would be impractical. The following versions are explicitly excluded from table generation: `2.0.0`, `2.1.2`, `3.0.0`, `3.1.0`, `3.2.0`, `4.0.0`.

## Decision
The supported Unicode versions, returned by `list_versions()`, are exactly (in ascending order):

`4.1.0`, `5.0.0`, `5.1.0`, `5.2.0`, `6.0.0`, `6.1.0`, `6.2.0`, `6.3.0`, `7.0.0`, `8.0.0`, `9.0.0`, `10.0.0`, `11.0.0`, `12.0.0`, `12.1.0`, `13.0.0`, `14.0.0`, `15.0.0`, `15.1.0`

This list is maintained in `wcwidth/unicode_versions.py` and generated by `bin/update-tables.py`. The list is a plain Python function returning a hardcoded list; it does not read files at runtime.

## Consequences
- New Unicode versions require running the code generation script and committing updated table files.
- The function is importable as part of the public API.
