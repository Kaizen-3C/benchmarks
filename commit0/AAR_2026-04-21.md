# After Action Review — commit0-lite Baseline Campaign

**Date:** 2026-04-21
**Duration:** ~18 hours of elapsed work over 2026-04-20/21
**Participants:** Project owner + Claude (Opus 4.7)
**Disposition:** Successful. Four baselines measured, one partial. Post-campaign data supports the ADR-0060 amendment and sharpens the Kaizen-delta narrative.

## Mission

Measure single-shot and iterative baselines on commit0-lite (16 libs) across Sonnet 4.6 and GPT-5.4, including a reproduction of OpenHands V1's published agent result. Establish the bar Kaizen-delta must clear.

## What worked (sustain)

**1. WSL pivot after 3 Windows-host bugs.** ADR-0060 originally mandated WSL; when we hit commit0's Windows-path bugs in the first hour, pivoting to WSL rather than patching Windows-side was the right call. We wasted ~90 min on the patches, but the pivot took 15 min once decided and unlocked everything downstream.

**2. Incremental aggregate writes.** Every sweep script writes `aggregate_*.json` after each library completes. When the initial B2 Sonnet sweep hit rate-limit connection drops on 14 of 16 libs, we already had wcwidth + deprecated's data persisted and didn't lose them. Same pattern saved us on two other mid-run failures.

**3. Dual-provider testing.** The Sonnet-only data would have supported "Reflexion regresses Sonnet" as a universal claim. Adding GPT-5.4 flipped the sign — Reflexion on GPT-5.4 *helps* — which is both more interesting and more honest. Cheap addition, high narrative value.

**4. Re-parsing from on-disk truth, not subprocess stdout.** Several times our runner's summary field was garbage (got the wrong line of pytest output, or captured httpx logs). Re-reading `test_output.txt` from the commit0 logs dir gave us the authoritative summary every time. Every per-lib JSON went through this hygiene pass.

**5. Cache counterfactual on B6 data we did have.** Once we had 3 instances of B6 Sonnet data (even though we stopped the sweep), the counterfactual analysis gave us a publishable "76% of the cost gap to OpenHands' published $1.88/lib is missing prompt caching" claim — without running B6 to completion.

**6. Running B3 and B6 GPT-5.4 in parallel.** After Sonnet work, the GPT-5.4 column filled in one evening with both sweeps in flight simultaneously. No interference; different venvs, different providers.

## What didn't work (improve)

**1. $85 in unmonitored B6 spend from a one-line field-name mismatch.** Our cost monitor read `total_cost` from the OpenHands output.jsonl; the actual field is `accumulated_cost`. The cap never fired. Three completed instances cost $10.45, $30.99, $43.26 with zero ticks from the monitor. This is the single biggest process defect of the campaign.

> **Lesson:** Cost monitors need integration testing — a smoke test that runs one instance, reads back its cost, and asserts `>0`. Any field-name drift would have surfaced in seconds instead of $85 later.

**2. Four cascading install failures before B6 produced any LLM call.** In order: (a) `uv run` workspace-source resolution, (b) SDK strict workspace-root check, (c) `RUNTIME_API_KEY` missing (OpenHands' hosted-runtime default), (d) `DockerDevWorkspace` validator bug in v1.11.5. Each was a ~15–30 min fix. Cumulatively ~2 hours.

> **Lesson:** Before trying to pin to a specific SDK version ("reproduce 7/16 exactly"), try the latest release first. If it works, great. If it doesn't, THEN you know you're fighting the SDK itself, not version drift. We did this backwards — pinned first, hit v1.11.5-specific bug, then switched to latest and it worked.

**3. "pip install commit0" timed out in 13 of 16 parallel containers.** 16 workspaces × heavy PyPI fetch saturated the host's outbound bandwidth. Reducing `--num-workers` from 16 to 3 for B6 GPT-5.4 made all 3 succeed.

> **Lesson:** Workspace-bootstrap parallelism is bandwidth-bound, not CPU-bound. `--num-workers` should scale with measured bandwidth, not core count. Default should probably be 4 for a local Docker environment.

**4. Sonnet-first scripts needed cloning for OpenAI.** `single_shot_sonnet.py`, `reflexion_sonnet.py` were written hardcoded to Anthropic. Adding GPT-5.4 required ~30 min of script-cloning + param adjustments (`max_completion_tokens` not `max_tokens`, OpenAI auto-cache vs Anthropic manual cache_control).

> **Lesson:** Parameterize provider from day one. A `--provider anthropic|openai` flag + a small LLM-client factory would have saved the fork. Extra complexity upfront = less duplicated code forever.

**5. v1.11.5 pin attempt ate 2+ hours, produced zero usable data.** The published 7/16 was SDK v1.11.5 + OpenHands' hosted RUNTIME_API; strict reproduction required both. The hosted runtime needs an internal API key we don't have. Without it, v1.11.5's local-Docker code path has a Pydantic validator bug that rejects valid input.

> **Lesson:** "Reproduce their exact pin" is often a misleading goal when the pin requires infrastructure you don't have. Cite their published number, run latest SDK, label honestly. The comparison is apples-to-apples at the architecture level; the version delta is a footnote.

**6. Reflexion baseline was catastrophically ineffective on Sonnet.** −19 pp aggregate, +50% cost, +50% wall time. pyjwt went from 133 passed → 1 passed because Sonnet's reflection rewrote working code. wcwidth went 29 → 1 in the exact same pattern.

> This isn't a *process* mistake — Reflexion was a planned baseline and its failure is part of the data. Lesson is more architectural: **"iterate until better" without grounding is not iteration quality, it's regression risk.** Kaizen-delta's iteration must ground each step in verifiable signal (tests pass/fail at module level), not on Sonnet's narrative summary of "what went wrong."

## Root causes

| # | Surface bug | Root cause |
|---|---|---|
| 1 | commit0 harness broken on Windows (3 bugs) | commit0 is Linux-first; Windows is a never-exercised code path |
| 2 | v1.11.5 DockerDevWorkspace validator | v1.11.5 local-Docker path was never exercised in OpenHands' own benchmark runs (they use hosted runtime) |
| 3 | Cost monitor field name mismatch | No smoke test for the monitor |
| 4 | pip-install timeouts in B6 | Default worker count too high for local bandwidth |
| 5 | Reflexion regression on Sonnet | Reflection-without-grounding relies on the model's self-assessment; Sonnet's is untrustworthy |

## What we learned about the subject matter

1. **Single-shot GPT-5.4 is the cheap-quality frontier.** $0.0038/test beats all four other Sonnet/Reflexion configurations.
2. **Reflexion is model-dependent.** -19 pp on Sonnet, +5 pp on GPT-5.4. Narrative implication: any future paper citing Reflexion results must name the model.
3. **Caching explains ~76% of OpenHands V1 cost overhead.** Their published $1.88/lib is a caching-dependent claim. Any architecture without aggressive caching will cost 4–15× more, same quality.
4. **Five libs fail across all four baselines** (chardet, marshmallow, babel, jinja, minitorch). These are the real signal test for Kaizen-delta. If decomposition can't move these off the floor, decomposition's value claim is weaker.
5. **Architecture dominates model at the agent-loop level.** B6 Sonnet and B6 GPT-5.4 got identical per-instance outcomes (same 2/3 resolved, same unresolved lib). Model swap only changed cost and speed.

## Paper readiness

After this campaign, we can write these claims with data-backed confidence:

1. Single-shot LLMs clear 40–51% of commit0-lite at $0.004–0.007/test. *(B2 data)*
2. Naive reflection regresses one frontier model (Sonnet −19 pp) and improves another (GPT-5.4 +5 pp) on the same benchmark. *(B3 data)*
3. Agent-loop architectures (OpenHands V1) reach 66% instance-resolution on completed libraries but without prompt caching cost 15× more than published. *(B6 partial + cache counterfactual)*
4. At the agent-loop level, problem solvability is architecture-bound, not model-bound. *(B6 Sonnet vs GPT-5.4 on same 3 libs)*
5. A cluster of commit0-lite libraries (chardet, marshmallow, babel, jinja, minitorch) is unsolvable by any baseline tested here. *(Kaizen-delta must move ≥1 of these to establish value)*

These five claims define the paper's findings section. Kaizen-delta's experiment, when run, either strengthens them ("decomposition closes the floor libs gap") or adds a sixth ("decomposition fails to beat single-shot GPT-5.4 efficiency" — still publishable, still honest).
